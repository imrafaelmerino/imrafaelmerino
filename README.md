- [json-values](https://github.com/imrafaelmerino/json-values) 

<a href="https://github.com/imrafaelmerino/json-values"> <img src="logos/json_values_150x86.png" alt="json-values"/> </a>

This was my very first project as a functional developer. I learned from Rich Hickey how important persistent data 
structures are. By the time I started developing json-values I was working with Json all the time. I really missed 
a persistent Json and a better API to manipulate them. I put into practice a lot new concepts like recursion, 
tail-recursive calls, trampolines, high order functions, functors, monads etc. 

I started testing json-values with a Scala library called ScalaCheck, that is used for property-based testing (PBT). 
This way I kill two birds with a stone; I learned a new language, Scala, and a new testing
philosophy. I discovered property-based testing thanks to John Huge. PBT led me to creating a beautiful Json generator. 
I challenged the world claiming it's the best Json generator in the whole wide world! You can benefit from this
Json generator to test your libraries. 

Spec is an incredible Clojure library. I implemented a json-spec based on the ideas I learned from Spec. 
At the risk of being presumptuous, I claim it's by far the best way of validating a Json in Java and Scala. 
It's easy to write, read and maintain. Defining json generators and specs is a breeze. 

Developing json-values I started taming effects with FP. I experienced myself how great FP is 
describing, composing and executing programs made up of effectful functions. I designed an interactive program
that creates a Json asking the user for every value on the console. I also created a Json future.

I developed json-values in Scala and Dotty as well. Dotty is the prerelease of Scala3.

  - [Java version](https://github.com/imrafaelmerino/json-values)
  - [Scala version](https://github.com/imrafaelmerino/json-scala-values)
  - [Dotty version](https://github.com/imrafaelmerino/json-dotty-values)
  
Scala is by far the best option to do FP in the JVM. I've learned a lot from the Scala community.
I have a tremendous respect for the Scala creator, Martin Ordesky, and the ZIO creator, John De Goes. I
never miss any of their talks and articles.

Clojure is a great alternative. It's really challenging since it's a lispy programming language. As 
a Lisp fan, being able to use it in the JVM is incredible. You must see any talk from Rich Hicky. 
He made a big impact on me, and the changed completely my life as a programmer.
